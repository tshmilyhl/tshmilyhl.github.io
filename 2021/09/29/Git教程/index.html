工作区和暂存区

工作区 (Working Directory)

就是电脑上你能看到的目录，比如当前的learngit文件夹就是一个工作区

版本库 (Repository)

工作区有一个隐藏目录**.git**,这个不算是工作区，而是Git的版本库
Git的版本库里存了很多东西，其中最重要的就是state(或者叫index)的暂存区，还有Git为我们自动创建了一个分支master,以及指向master的一个指针HEAD。

把文件往Git版本库添加的时候，是分两步执行的:

第一步使用git add把文件添加进去，实际上就是把文件添加到暂存区；

第二步使用git commit 提交更改，实际上就是把暂存区的内容提交到当前分支。

因为我们创建Git版本库时，Git自动为我们创建了唯一一个master分支，所以，现在git commit 就是往master分支上提交更改。

你可以理解，需要提交的文件修改通通放到暂存区，然后，一次性提交暂存区的所有修改。

删除文件

情况一:工作区文件删除，无其他操作

1.rm test.txt

以下命令可恢复文件

1.git restore test.txt （新git版本）

2.git checkout – test.txt (理解checkout)

情况二:工作区文件删除，版本库文件删除

1.rm test.txt

2.git rm test.txt

3.git commit -m “remove test.txt”

想要恢复文件的话，只能参考版本回退的方式:

git reset –hard HEAD^

从来没有被添加到版本库就被删除的文件，是无法恢复的
